About Me

처음엔 화면을 만들었고, 지금은 그 화면 뒤의 모든 것을 만듭니다.
퍼블리셔로 시작해 백엔드 개발자로 전환했고, IoT 플랫폼 2.0 개발팀에 합류해 모듈 개발을 담당했습니다. 이후 담당자 이동으로 AWS 인프라 운영까지 맡으면서, 화면부터 서버와 인프라까지 서비스 전체 흐름을 직접 경험할 수 있었습니다.
한 영역에 머물지 않고 필요한 곳으로 역할이 넓어진 경험이, 시스템을 넓게 바라보는 시각을 만들어줬다고 생각합니다.

개발할 때 기능 자체보다 "왜 필요한가"와 "이 결정이 어디까지 영향을 미치는가"를 먼저 생각하려 합니다.
좁은 범위에서 구현을 완성하는 것보다, 전체 흐름 안에서 이 기능이 어디에 위치하는지를 파악하고 사이드이펙트를 미리 고려하는 것이 습관이 됐습니다. 문제가 생겼을 때도 증상을 빠르게 덮기보다 근본 원인을 찾는 방향으로 접근합니다.
아직 부족한 점이 많지만, 무언가를 만들기 전에 한 번 더 "왜"를 묻고 "그래서 어디까지 영향이 가는가"를 생각하는 태도, 그 습관만큼은 계속 가져가고 싶습니다.

Security
OAuth2 JWT 이중화 이슈
(재기동 시 로그아웃)
Problem

서버 재기동마다 키 재생성으로 인해 Sticky Session에 의존하게 되며, 무중단 배포 시 전체 사용자가 로그아웃되는 현상 발생

Solution

DB에 RSA 키를 생성·저장하고 기동 시 DB에서 조회하는 방식으로 전환하여, 서버 재기동 시에도 키 일관성 유지 및 세션 유지 보장

CloudFront 이미지 캐시
(교체 후 미반영)
Problem

S3에 동일 파일명으로 이미지를 교체 업로드해도 CloudFront가 기존 캐시를 계속 반환하여 변경된 이미지가 반영되지 않음

Infra
EC2 스케줄 부팅 실패
(CONSOLE_EVTCHN 오류)
Problem

평일 09:00 자동 시작 스케줄 운영 중, t2(Xen) 기반 인스턴스에서 커널-하이퍼바이저 콘솔 채널 협상 타이밍 이슈로 간헐적 부팅 실패 발생. 직렬 콘솔 미지원으로 내부 접근 불가

AWS IoT
OTA 디바이스 상태 반영 지연
(EventBridge 폴링 구조)
Problem

EventBridge 5분 주기 폴링으로 최대 5분 지연 발생. 진행 중인 Job 없어도 무조건 실행되어 불필요한 Lambda 호출 비용 낭비. 상태 전이 가드 부재로 SUCCEEDED 이후 IN_PROGRESS 덮어쓰기 역전 가능성 존재

Backend
GlobalExceptionHandler 미처리

Problem
